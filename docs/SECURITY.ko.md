## 보고 범위(In scope)

- **`rtrt-proxy` / `rtrt-cli`의 커맨드 인젝션** — 검증되지 않은 명령 출력 / 인자를 셸로 흘리는 경로.
- **`rtrt-templates`의 경로 우회** — 매니페스트가 대상 디렉터리 밖으로 파일을 쓰는 경우.
- **`rtrt-memory`의 SQL 인젝션** — 신뢰되지 않은 문자열이 FTS5 `MATCH`나 다른 SQL 절에 파라미터 바인딩 없이 흘러 들어가는 경우.
- **자격 증명 노출** — API 키, 베어러 토큰, 메모리 내용이 로그/응답/MCP 도구 호출로 새는 경우.
- **대시보드 SSRF** — 신뢰되지 않은 입력으로 외부 호출을 일으키는 HTTP 엔드포인트.
- **대시보드/MCP의 DoS** — 사소한 페이로드로 크래시되거나 메모리가 폭증하는 경우.

## 범위 외(Out of scope)

- 호스트에 물리적으로 접근이 필요한 공격
- 사회공학 공격
- 서드파티 의존성 자체의 취약점(해당 프로젝트에 보고하고, RTRT는 패치된 버전으로 추적/업데이트)
- `--bind 0.0.0.0`이 필요해야 일어나는 DoS (대시보드 기본은 `127.0.0.1`)

## 보안 베스트 프랙티스

- **파라미터화된 SQL만**: `rtrt-memory`는 `rusqlite::params!`만 사용합니다.
- **경계에서 허용 목록 정규식**: 템플릿 이름, 명령 이름 등 파일시스템 경로 컴포넌트로 쓰이는 문자열은 사용 전 검증합니다.
- **argv 리스트만**: `rtrt-templates` 포스트-인스톨 훅은 명시적 argv 리스트로 명령을 실행하며, 호출 측에서 `shell=true` 같은 형태를 사용하지 않습니다.
- **코드와 git에 시크릿 금지**: API 키, 베어러 토큰, 자격증명이 커밋되지 않습니다. 기본 `.gitignore`가 `.env`, `*.pem`, `*.key` 그리고 AI 도구 임시 파일을 차단합니다.
- **기본 로컬 바인딩**: `rtrt-dashboard`는 `127.0.0.1`, `rtrt-mcp`는 stdio가 기본. 원격 노출은 옵트인.
- **텔레메트리 없음**: RTRT는 사용 통계를 수집하지 않습니다. 대시보드의 절감 통계는 로컬 데이터로만 계산됩니다.

## 프로바이더 자격증명

채팅 프로바이더(Anthropic, OpenAI, OpenAI 호환)의 API 키는 환경 변수에서 읽으며 RTRT 자체가 디스크에 쓰지 않습니다. 대시보드는 자격증명을 노출하는 엔드포인트를 제공하지 않습니다.

자격증명이 로그되거나 응답에 나타나거나 디스크에 남는 사례를 발견하면 보안 버그로 간주해 위 절차로 신고해 주세요.
